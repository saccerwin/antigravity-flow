# linkedin-technical

**B2B social skill** — Reach developers and engineering leaders on LinkedIn.

## What It Does

Guides you through LinkedIn for technical audiences: content that works, reaching engineering managers, company page vs personal, formatting, and when LinkedIn beats Twitter.

## When to Use

- Creating technical content for LinkedIn
- Reaching B2B decision-makers (CTOs, VPs, EMs)
- Building thought leadership for dev tools
- When you say "LinkedIn," "B2B developer marketing," or "reaching engineering managers"

## Quick Start

```
/linkedin-technical

I want to announce our open source project on LinkedIn
in a way that reaches engineering managers at
enterprises. How should I structure the post?
```

## Key Sections

1. **When LinkedIn Beats Twitter** — Understanding the platform
2. **Content That Works** — Formats and formulas
3. **Formatting Best Practices** — Hooks, structure, visuals
4. **Company Page vs Personal** — When to use each
5. **Reaching Engineering Managers** — What EMs care about
6. **Profile Optimization** — Technical professional profile

## When to Use LinkedIn vs Twitter

| Use LinkedIn | Use Twitter |
|--------------|-------------|
| Selling to enterprises | Individual developers |
| Reaching eng managers | Open source projects |
| B2B partnerships | Real-time engagement |
| Recruiting | Casual/authentic voice |
| Longer professional content | Technical hot takes |

## Content That Works

| Type | Performance |
|------|-------------|
| Personal stories | Very high |
| Career lessons | High |
| Contrarian opinions | High |
| Behind-the-scenes | High |
| Technical deep dives | Medium |
| Plain announcements | Low |
| Shared links | Very low |

## Post Structure

1. Hook (first 210 chars critical)
2. Context or problem
3. Bullet points (3-5 insights)
4. Personal take
5. Question CTA
6. 3-5 hashtags at end
