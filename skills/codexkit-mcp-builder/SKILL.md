# codexkit-mcp-builder

Treat MCP server work as product design for AI agents. Focus on high-value workflows, concise response shapes, and tool contracts that are easy for a model to use correctly under context pressure. When planning or implementing an MCP server, cover the research phase, tool selection, shared infrastructure, validation schemas, error handling, and an evaluation loop that tests realistic tasks. Support both Python and TypeScript implementations, but keep the guidance centered on agent usability rather than SDK boilerplate.

## Capabilities

- agent-centric-tool-design
- mcp-server-planning
- python-or-typescript-mcp-guidance
- evaluation-driven-tooling




## Expected Output

- Design MCP tools around user workflows instead of mirroring raw API endpoints one-for-one.
- Call out input validation, response shaping, pagination, and error-message quality as first-class concerns.
- Include a practical evaluation loop with representative questions or scenarios so the server can be validated by agent behavior.
- Differentiate planning guidance from implementation guidance for Python and TypeScript stacks.
