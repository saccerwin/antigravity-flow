# codexkit-codebase-review

Review large codebases by chunking them into modules or slices that fit the context window. Inspect each chunk for local defects, then synthesize cross-cutting issues across boundaries, shared contracts, and rollout risks. Use this when a full sweep is needed instead of a small diff review.
