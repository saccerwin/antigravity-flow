# Pixel Parity Checklist (SwiftUI)

- Source includes all target breakpoints and states.
- Spacing values match design spec (outer padding, inter-item gaps, section spacing).
- Typography matches font family, size, weight, and line height intent.
- Color and opacity values match tokens or approved equivalents.
- Radius, border width, and shadow match design.
- Icon size and alignment match design baseline.
- Touch target and interaction states are complete (pressed, focused, disabled, loading).
- Dynamic Type behavior is acceptable and does not break layout.
- VoiceOver labels and ordering are present for interactive elements.
- Side-by-side visual diff reviewed and final deltas are zero or explicitly approved.
