# Kỹ năng Quản lý Sản phẩm

> **Dành cho Product Managers, Product Owners và Product Analysts.** Viết spec tính năng, lập roadmap, tổng hợp nghiên cứu người dùng, và cập nhật stakeholders nhanh hơn.

## Kỹ năng con

| Kỹ năng | Mục đích | Kích hoạt khi |
|---------|----------|---------------|
| `viết-spec` | Problem statement, user stories, acceptance criteria | "viết spec cho", "định nghĩa tính năng" |
| `cập-nhật-roadmap` | Cập nhật roadmap với ưu tiên mới | "cập nhật roadmap", "ưu tiên tính năng" |
| `sprint-planning` | Scope sprint, story points, capacity | "lên kế hoạch sprint", "sprint review" |
| `tổng-hợp-nghiên-cứu` | Tổng hợp phỏng vấn, khảo sát, dữ liệu | "tổng hợp research", "insights người dùng" |
| `phân-tích-đối-thủ` | So sánh tính năng, phân tích định vị | "so sánh đối thủ", "competitive brief" |
| `review-metrics` | Theo dõi KPIs, phân tích funnel, cohort | "review metrics", "phân tích sản phẩm" |
| `cập-nhật-stakeholders` | Tóm tắt điều hành, báo cáo tiến độ | "cập nhật stakeholders", "báo cáo tiến độ" |

## Ví dụ sử dụng

```
"Viết spec cho tính năng thanh toán bằng QR code"
→ Agent thu thập requirements, viết problem statement, liệt kê user stories,
   xác định success metrics, đề xuất acceptance criteria
```

## Workflow liên quan

- `/product-management-write-spec` — Tài liệu spec tính năng đầy đủ
- `/product-management-roadmap-update` — Cập nhật ưu tiên roadmap
- `/product-management-sprint-planning` — Scope và kế hoạch capacity sprint
- `/product-management-synthesize-research` — Tổng hợp kết quả nghiên cứu
- `/product-management-competitive-brief` — So sánh tính năng đối thủ
- `/product-management-metrics-review` — Review KPI sản phẩm
- `/product-management-stakeholder-update` — Tóm tắt tiến độ cho lãnh đạo
