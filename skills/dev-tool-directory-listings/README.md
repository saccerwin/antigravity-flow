# Developer Tool Directory Listings

Getting listed on developer tool directories and GitHub awesome lists.

## What This Covers

- Identifying which directories matter for developer tools
- GitHub awesome-list submission strategy
- Writing effective directory descriptions
- Maintaining listings over time
- Tracking referral traffic and attribution

## Key Concepts

**Developer directories differ from business directories** - they're often community-maintained, have real editorial standards, and developers actually use them for tool discovery.

**Awesome lists require authentic contribution** - build credibility by contributing value, not just promoting your own tool.

**Quality over quantity** - ten high-value listings beat a hundred spam directories.

## When to Use

- Launching a developer tool and building discovery channels
- Planning directory submission strategy
- Contributing to GitHub awesome-lists
- Auditing and maintaining existing listings

## Quick Start

1. Identify 3-5 awesome-lists in your category
2. Read contribution guidelines thoroughly
3. Prepare assets (logo, screenshots, descriptions)
4. Submit PRs following exact guidelines
5. Set up referral tracking
6. Review and maintain quarterly
