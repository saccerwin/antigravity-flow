# Code Review

## When to Use
Full PR lifecycle — request reviews, handle feedback, complete branch integration.

## Procedure
1. Clarify the problem, constraints, and success criteria.
2. Reproduce or prove the issue before changing code.
3. Apply the smallest safe change, then verify with tests or evidence.

## Expected Output
- Clear implementation or review guidance
- Validation evidence
- Any follow-up risks or regressions
